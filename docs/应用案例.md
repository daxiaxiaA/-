# 应用案例

## 案例名称

课程实验文件的离线安全传输演示

## 应用场景

发送方需要把一份实验报告或文本文件发送给接收方。发送方希望接收方能够确认两个事实：

1. 文件内容在传输过程中没有被篡改。
2. 文件确实由持有发送方私钥的一方完成签名。

## 使用方式

1. 发送方 A 启动程序。
2. 选择“文件”输入模式，并选择待传输文件。
3. 选择 AES 作为对称加密算法，选择 SHA-256 作为 Hash 算法，选择 2048 位 RSA 密钥。
4. 生成 A/B 两组 RSA 密钥对。
5. 点击“执行完整流程”。
6. 程序输出签名、加密、密钥封装、解密和验签日志。
7. 点击“保存密文包”，得到 `.crypto` 文件。
8. 接收方 B 打开密文包，程序解密并输出验签结果。

## 使用效果

程序能够还原原始文件内容，并在日志中输出“验签结果：通过”。如果密文包内容被破坏，打开或验签过程会失败，从而提示文件可能被篡改或密钥不匹配。

## 价值说明

该案例适用于密码学、网络安全、信息安全课程的实验演示，也适用于解释混合加密、数字签名和完整性校验之间的关系。
