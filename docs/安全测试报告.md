# 安全测试报告

## 测试目标

验证软件不存在人为植入的恶意行为，并确认核心签名、加密、解密和验签流程可以按预期完成。

## 依赖与外连说明

- 项目只使用 JDK 标准库。
- 项目无第三方依赖。
- 程序无网络通信代码，不主动连接互联网。
- 程序只在用户主动选择文件、保存密文包或保存解密结果时访问本地文件系统。

## 自检命令

构建：

```bash
./build.sh
```

自检：

```bash
java -jar release/CryptoTaskApp.jar --self-test
```

如系统默认 Java 不是 JDK 17，可使用：

```bash
/usr/lib/jvm/java-17-openjdk-amd64/bin/java -jar release/CryptoTaskApp.jar --self-test
```

## 当前测试结果

测试时间：2026-06-06

本机默认 `java` 为 Java 8，无法运行 JDK 17 编译的 jar；使用 JDK 17 路径运行自检通过。

通过用例：

- AES + SHA-256
- AES + SHA-1
- AES + MD5
- DES + SHA-256
- DES + SHA-1
- DES + MD5

每个用例验证内容：

1. 生成 A/B 两组 RSA 密钥。
2. 生成或派生对称密钥 `K`。
3. 计算明文 Hash。
4. 使用 A 私钥生成签名。
5. 使用 AES 或 DES 加密 `M || 签名`。
6. 使用 B 公钥加密 `K`。
7. 使用 B 私钥解出 `K`。
8. 解密 `M || 签名`。
9. 使用 A 公钥解出签名摘要。
10. 重新计算明文 Hash 并比较。
11. 编码密文包，重新解码密文包，再次解密验签。

## 恶意代码排查

源码中未发现以下行为：

- 后门账户或隐藏认证逻辑。
- 网络上传、下载、远程命令执行。
- 窃取本地文件、环境变量或系统凭据。
- 删除、覆盖非用户选择路径的文件。
- 动态加载远程代码。

## 风险说明

本项目用于课程实验和流程演示。DES、MD5、SHA-1 已不适合作为现代生产系统的首选安全算法，但任务要求中需要支持 DES 和 MD5/SHA，因此程序保留这些算法作为可选项。实际生产环境建议优先使用 AES 与 SHA-256，并采用标准签名算法和密钥管理方案。
