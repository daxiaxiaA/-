## 功能介绍

数字签名与加密传输软件实现了发送方 A 签名并加密、接收方 B 解密并验签的完整流程。软件支持 AES、DES 两种对称加密算法，支持 SHA-256、SHA-1、MD5 三种 Hash 算法，支持生成 A/B 两组 RSA 公私钥。软件既可以处理字符串明文，也可以处理文件内容，并支持保存和重新打开 `.crypto` 密文包。

软件包含以下独立功能：

1. AES/DES 对称加密。
2. SHA-256/SHA-1/MD5 摘要计算。
3. RSA 公私钥生成。
4. 使用发送方私钥生成数字签名。
5. 使用接收方公钥加密对称密钥。
6. 接收方解密并验签。
7. 字符串与文件两种输入模式。
8. 密文包保存和重新验证。
9. 命令行自检。

软件可编译为 `release/CryptoTaskApp.jar`，具备 Swing 可视化用户界面。

## 社会价值

本项目面向密码学、网络安全和信息安全基础教学，能够直观展示 Hash、数字签名、对称加密、非对称加密和混合加密之间的关系。与只展示单一算法的程序不同，本项目把发送方签名加密、信道传输和接收方解密验签整合在一个可视化程序中，便于学习者理解真实安全通信协议中的关键步骤。

项目只依赖 JDK 标准库，部署简单，可在普通教学机房环境中运行。项目也可以作为其他课程实验、演示工具或教学样例的基础代码使用。

## 文档说明

项目已提供 `README.md`，包含环境要求、构建方式、运行方式、自检方式、使用步骤和目录结构说明。项目要求使用 JDK 17 构建和运行。构建命令为：

```bash
./build.sh
```

运行命令为：

```bash
java -jar release/CryptoTaskApp.jar
```

自检命令为：

```bash
java -jar release/CryptoTaskApp.jar --self-test
```

## 安全测试

项目提供命令行自检功能，覆盖 AES/DES 与 SHA-256/SHA-1/MD5 的 6 种组合。自检会验证生成密钥、签名、加密、密钥封装、解密、验签、密文包编码和密文包重新打开。

当前测试结果：使用 JDK 17 运行自检通过，输出 `SELF_TEST_PASSED`，共通过 6 个算法组合。

项目无第三方依赖，无网络外连代码，不包含远程命令执行、隐藏账户、后门逻辑或非用户授权的文件删除逻辑。
